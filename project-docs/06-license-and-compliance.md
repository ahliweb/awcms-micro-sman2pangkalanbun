# License and Compliance Guide

## Selected License

This project is intended to use the **AW Non-Commercial License**:

- Reference repository: `https://github.com/ahliweb/aw-non-commercial-license`
- License text page: `https://github.com/ahliweb/aw-non-commercial-license?tab=License-1-ov-file`

## Compliance Intent

- Keep this repository and its derivative deliverables aligned with non-commercial usage terms.
- Prevent accidental commercialization without explicit re-licensing.
- Maintain visible documentation of licensing boundaries for contributors.

## Operational Controls

- Add explicit license notice in root-level project documentation.
- Require PR checklist confirmation for license-safe changes.
- Flag integrations that may imply paid/commercial distribution.
- Track third-party dependency licenses before production release.

## Contributor Guidance

- Contributors should read license terms before submitting major changes.
- Any feature that changes distribution or monetization posture requires a policy review issue.
- External assets (images, fonts, documents) must include usage rights verification.

## Release Checklist (License-Safe)

- Confirm current use case is non-commercial.
- Confirm no contractual monetization hooks are embedded.
- Confirm legal notices are present in site/footer/about or equivalent location.
- Confirm dependencies are compatible with project distribution model.

## Suggested Repository Additions

- Add `LICENSE` file containing or referencing AW Non-Commercial terms.
- Add `NOTICE.md` summarizing project license and usage boundaries.
- Add PR template checkbox for non-commercial compliance confirmation.

## Important Note

This document is operational guidance, not legal advice. For legal certainty, review terms directly in the official license repository and consult legal counsel where required.
